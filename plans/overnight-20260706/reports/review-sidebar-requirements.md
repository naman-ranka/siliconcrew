## Product-posture consistency
- **IDE vs agent posture:** density is layout emphasis only (same store, same components) — exactly the CLAUDE.md rule ("posture is layout emphasis only"). No command palette / file-creation leaks into agent view via these changes.
- **Runtime seam:** the Codex/native split is a clean conditional render at the composer (`ChatInput.tsx:151`), plus `data-runtime` theming (pre-existing). `ModelPicker` is no longer referenced anywhere except a comment (`AgentShell.tsx:359`) — no stray codex path left in the native picker. `selectThread` (`store.ts:1488-1495`) makes `agentRuntime` FOLLOW the active thread, so a URL-selected Codex chat gets the Codex picker/theme, never the native list against a Codex conversation.

## Low-severity observations (non-blocking)
1. **Initial-load runtime follow:** `selectThread` returns early when `threadId === activeThreadId` (`store.ts:1483`), so the runtime-follows-thread flip happens on *switches*. If a session's first-loaded active thread is a Codex thread set outside `selectThread`, confirm `agentRuntime` is seeded from it. The X2C-6 label bug is mitigated regardless (Codex picker falls back to `codexDefaultModel`), so this is cosmetic at worst — worth a quick manual check on staging.
2. **`CODEX_CATALOG` row duplication:** 3 rows mirror `CATALOG` entries. Intentional (documented) and priced-consistent, but a future model refresh must update both lists — the test (`test_model_selector.py`) guards pricing/provider but not label drift between the two.
